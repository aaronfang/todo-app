# 2026-03-12

## 修复窗口尺寸自动调整问题

### 问题描述
用户反馈每次修改或新建任务后，窗口会缩小到默认高度，在任务列表较长时影响使用体验。

### 解决方案
1. 修改 `adjust_window_size()` 方法，增加 `allow_height_change` 参数（除了原有的 `allow_width_change`）
2. 修改 `populate_listbox_without_width_change()` 方法，改名实际上应该是"保持窗口尺寸不变"，现在同时保持宽度和高度
3. 在以下操作中使用 `populate_listbox_without_width_change()` 来保持窗口尺寸：
   - `add_task()` - 添加新任务
   - `edit_task()` - 编辑任务/分隔符
   - `add_separator_title()` - 添加分隔符标题
   - `add_separator_below()` - 添加分隔符
   - `remove_selected_tasks()` - 删除任务（后续补充）
   - 所有任务状态变更操作（完成、取消、紧急等）

### 保留自动调整的场景
- 初始加载 (`populate_listbox()`)
- 字体大小调整 (`update_font_size()`)
- 窗口首次显示

### 效果
用户可以自由调整窗口大小，之后的任务操作都会保持用户设置的窗口尺寸，不会再自动缩小。
